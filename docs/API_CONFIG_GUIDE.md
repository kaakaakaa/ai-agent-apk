# AI Agent - API 配置和测试指南

## 新增功能

### 1. API 连接测试
现在您可以在设置界面中测试 API 连接是否正常，避免配置错误导致执行失败。

### 2. 可选模型选择
支持从厂商获取可用模型列表，并选择您想要使用的模型。

### 3. API 连通性判断
自动检测 API Key 是否有效，模型是否可用。

## 使用步骤

### 第一步：获取 API Key

#### 阿里云百炼
1. 访问 https://bailian.console.aliyun.com/
2. 登录阿里云账号
3. 进入「API-KEY 管理」页面
4. 点击「创建新的 API-KEY」
5. 复制生成的 API Key（以 `sk-` 开头）

#### DeepSeek
1. 访问 https://www.deepseek.com/
2. 登录后进入「个人中心」
3. 选择「API Keys」
4. 创建并复制 API Key

#### Kimi 月之暗面
1. 访问 https://kimi.moonshot.cn/
2. 登录后进入「控制台」
3. 选择「API Key 管理」
4. 创建并复制 API Key

### 第二步：配置 API Key

1. 打开应用
2. 点击底部 **⚙️ 设置** 按钮
3. 选择大模型厂商（推荐阿里云百炼）
4. 输入 API Key
5. 点击 **🔍 测试 API 连接** 按钮
6. 等待测试结果：
   - ✓ 绿色文字表示连接成功，会显示可用模型列表
   - ✗ 红色文字表示连接失败，会显示错误信息
7. 如果连接成功，从下拉框选择想要的模型
8. 点击 **💾 保存 API Key** 按钮

### 第三步：验证配置

1. 返回主界面
2. 查看左上角状态：
   - **准备就绪 + 云端模型**：表示 API Key 配置成功，可以正常使用
   - **未配置 API Key**：需要重新检查配置
   - **未就绪**：可能是初始化问题，尝试重新打开应用

### 第四步：开始使用

在主界面：
- **输入框**：输入你想让 AI 帮你做的事
- **发送按钮**：发送指令
- **快捷按钮**：快速执行常用操作（打开微信、返回等）

#### 示例指令
- "打开微信"
- "返回"
- "打开设置"
- "帮我发一条微信消息给张三说你好"

## 常见问题

### Q: 测试 API 连接失败？
A: 可能原因：
1. API Key 输入错误（检查是否有空格）
2. 网络连接问题（检查手机是否联网）
3. API Key 已过期或被禁用（重新创建）
4. 选择的厂商不正确（确认 API Key 对应的厂商）

### Q: 显示"准备就绪"但执行失败？
A: 可能原因：
1. 无障碍服务未启动（检查设置 → 无障碍）
2. 模型选择不兼容（尝试切换到 qwen-turbo 或 qwen-plus）
3. API 调用超时（增加网络超时设置）

### Q: 执行成功但没有实际效果？
A: 这是正常现象，某些操作可能需要更长时间才能看到效果，或者当前应用不支持该操作。

## 支持的模型

### 阿里云百炼
- `qwen-turbo` - 速度快，成本低
- `qwen-plus` - 平衡性能和成本
- `qwen-max` - 最强性能，适合复杂任务

### DeepSeek
- `deepseek-chat` - 对话模型
- `deepseek-coder` - 代码专用模型

### Kimi
- `moonshot-v1-8k` - 8K 上下文
- `moonshot-v1-32k` - 32K 上下文
- `moonshot-v1-128k` - 128K 上下文

## 错误码说明

| 错误码 | 说明 | 解决方法 |
|--------|------|----------|
| 401 | 认证失败 | 检查 API Key 是否正确 |
| 403 | 权限不足 | 确认 API Key 有访问权限 |
| 404 | 模型不存在 | 检查模型名称是否正确 |
| 429 | 请求过多 | 稍后重试或升级配额 |
| 500 | 服务器错误 | 稍后重试 |

## 技术细节

### API Endpoint
- 阿里云百炼：`https://dashscope.aliyuncs.com/compatible-mode/v1`
- DeepSeek：`https://api.deepseek.com`
- Kimi：`https://api.moonshot.cn/v1`

### 认证方式
使用 Bearer Token 认证：
```
Authorization: Bearer YOUR_API_KEY
```

### 请求格式
采用 OpenAI 兼容格式：
```json
{
  "model": "qwen-max",
  "messages": [
    {"role": "system", "content": "你是一个 AI 助手"},
    {"role": "user", "content": "你好"}
  ]
}
```
