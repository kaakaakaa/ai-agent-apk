- 订阅 `ReActEngine.sessionState` StateFlow，在任务完成时取消监听

**代码变化**:

### 2. 修复"屏幕信息不可用"问题 ✅

**文件**:
- `CloudAgent.kt`
- `ReActEngine.kt`
- `AccessibilityScanner.kt`

**问题原因**:
- `buildContextMessage` 在 UI 树为 null 时直接返回"不可用"
- 没有重试机制获取 UI 树
- 无障碍事件监听不够全面

**修复方案**:

#### CloudAgent.kt
- 增强 `buildContextMessage` 方法
- 在 UI 树为 null 时尝试重新扫描
- 提供详细的错误信息和解决步骤

```kotlin
private fun buildContextMessage(uiTree: UiTree?): String {
    if (uiTree == null || uiTree.nodes.isEmpty()) {
        // 尝试重新扫描屏幕
        val freshUiTree = AccessibilityScanner.getUiTreeSync()
        if (freshUiTree != null && freshUiTree.nodes.isNotEmpty()) {
            return "当前屏幕内容：\n${freshUiTree.toJson()}"
        }
        // 提供详细的错误信息
        return buildString {
            appendLine("当前屏幕内容暂时不可用...")
            appendLine("当前活动应用：$packageName")
        }
    }
    return "当前屏幕内容：\n${uiTree.toJson()}"
}
```

#### ReActEngine.kt
- 添加 `getUiTreeWithRetry()` 方法
- 实现最多 3 次重试，每次间隔 200ms

```kotlin
private suspend fun getUiTreeWithRetry(maxRetries: Int = 3, retryDelayMs: Long = 200): UiTree? {
    for (i in 0 until maxRetries) {
        val uiTree = AccessibilityScanner.getUiTreeSync()
        if (uiTree != null && uiTree.nodes.isNotEmpty()) {
            return uiTree
        }
        if (i < maxRetries - 1) {
            kotlinx.coroutines.delay(retryDelayMs)
        }
    }
    return null
}
```

#### AccessibilityScanner.kt
- 增强 `onAccessibilityEvent` 监听更多事件类型
- 添加 `forceScanUiTree()` 方法供主动触发扫描
- 优化 `scanCurrentWindow` 错误处理

**文件**:

- `AgentOverlay.kt`

**修复方案**:

#### AgentService.kt
- 添加 `AgentOverlay` 实例管理
- 添加 `initializeAgentOverlay()` 方法
- 订阅 `ReActEngine` 状态并更新悬浮窗
- 在任务开始时显示悬浮窗，完成后隐藏

```kotlin
private fun initializeAgentOverlay() {
    agentOverlay = AgentOverlay(context)

    _reActEngine?.let { engine ->
        serviceScope.launch {
            engine.currentStep.collect { step ->
                step?.let {
                    agentOverlay?.updateStatus(it, isComplete = it.isComplete)
                }
            }
        }
    }
}
```

#### AgentOverlay.kt
- 优化 `updateStatus` 方法，避免重复查找视图

## 修复日志

### 2026-03-28 11:05 - 修复 AgentOverlay 主题错误

**问题**: 悬浮窗崩溃，错误信息：
```
java.lang.IllegalArgumentException: The style on this component requires your app theme to be Theme.MaterialComponents
```

**原因**: `AgentOverlay` 使用系统上下文创建悬浮窗时，没有应用正确的 MaterialComponents 主题。

**解决方案**: 在 `AgentOverlay.kt` 中使用 `ContextThemeWrapper` 包装上下文，应用 `Theme_AndroidAgent` 主题。

```kotlin
val themedContext = android.view.ContextThemeWrapper(context, R.style.Theme_AndroidAgent)
overlayView = LayoutInflater.from(themedContext).inflate(R.layout.agent_overlay, null)
```
